For repetition structures I would choose my precisionHeist game. In this game I had one method in particular that had a lot of complex repetition structures and integrated selection which made it very valuable. This function would be the guardControl function. This function is not that hard to look at, however it has a few very key and important parts that make it so useful to my game and it is the most important part.

Firstly what it does is, it sets a for loop that looks where to spawn in the guards. At the top of my code I have arrays written out that mention points like guardX, guardY, guardSpeed, if the guard is moving left or right, guardMinX, guardMaxX, guardMinY, guardMaxY, and guardTimings. At the top guardX and guardy have coordinates written within them, that when put into a for loopless than 5, spawns a guard at all 4 points. Then it checks what timing to place all the guards, by referencing the guardTimings array and to see if it passed that specific timing to run the loop again and spawn the next one in. After this it checks whether to stop the guards or not if the game is either won, or whether the player is caught.

Then it checks if the guard is going up or down, through the VertorHor boolean array, where true is vertical movement, and false is horizontal. If the movement is vertical, it adds the guard speed to the Y, until it hits the miny or maxy, which have their own array assigned to each guard, and then the guardSpeed is reversed by multiplying by -1. It is the same logic for the horizontal ones, just replacing maxY and minY, with maxX and minX. If the game is won, or the player is caught, the game runs the image of the guard and freezes it so that the game is over.

The issues I had with this logic is that it was very confusing to set up the for loop in a way where the guard speed wasn't causing issues. Before this I tried to use a single variable for guardSpeed, but it would reverse the guardSpeed for the entire row. This meant that one guard would hit the wall and reverse, but another guard would do the same regardless of whether it hit the wall or not. To fix this, I made an array, where all guards got the same speed, but it was individually assigned so that there were no mishaps.
